Get Device List ​

keyboard.getDevices()

Brief description: Gets the list of devices connected to the system.

Parameters ​

This method does not require any parameters.

Return Value ​

• Overall type: Promise<{ code: number; devices: Device[] }> • Description: Returns a Promise that resolves to an object containing the result code in code and the device list in devices. • Device structure inside devices:

FieldTypeDescriptionExample
dataobjectDevice data object-
idstringUnique device identifier"7338:2054"
usagenumberDevice usage value1
usagePagenumberDevice usage page value65440
vendorIdnumberVendor ID7338
productIdnumberProduct ID2054
productNamestringProduct name"GT60 PRO"

Example ​

javascript
import { ServiceKeyboard } from 'wvke-sdk';

async function fetchDevices() {
	const keyboard = new ServiceKeyboard();

	try {
		const { code, devices } = await keyboard.getDevices();
		if (code !== 0) {
			console.error('Failed to get device list, code:', code);
			return;
		}

		console.log('Discovered devices:', devices);
		if (devices.length > 0) {
			// Operate on the device list, for example initialize the first device
			// await keyboard.init(devices[0].id);
		} else {
			console.log('No devices detected.');
		}
	} catch (error) {
		console.error('Failed to get device list:', error);
	}
}

fetchDevices();

Initialize Specified Keyboard Device ​

keyboard.init()

Brief description: Initializes a specific keyboard device for subsequent interaction.

Parameters ​

FieldTypeDescriptionRequired
idstringUnique identifier of the device to initialize, available from the devices array returned by keyboard.getDevices()Yes

Return Value ​

• Overall type: Promise<{ code: number; info: string }> • Description: Returns a Promise that resolves to an initialization result object. • code: 0 means success, any other value means failure • info: initialization result message

Example ​

javascript
import { ServiceKeyboard } from 'wvke-sdk';

async function initializeFirstDevice() {
	const keyboard = new ServiceKeyboard();

	try {
		const { code, devices } = await keyboard.getDevices();
		if (code !== 0 || devices.length === 0) {
			console.log('No device available for initialization.');
			return;
		}

		const result = await keyboard.init(devices[0].id);
		if (result.code === 0) {
			console.log('Device initialized successfully:', result.info);
			// const deviceInfo = await keyboard.getDevicesInfo();
		} else {
			console.log('Device initialization failed:', result.info);
		}
	} catch (error) {
		console.error('Error while initializing device:', error);
	}
}

// initializeFirstDevice();

Get Device Information ​

keyboard.getDevicesInfo()

Brief description: Gets information for the currently initialized device.

Parameters ​

This method does not require parameters and is usually used after a successful call to keyboard.init().

Return Value ​

• Overall type: Promise<DeviceInfo> • Description: Returns a Promise that resolves to a DeviceInfo object containing device information. • Resolved object structure (DeviceInfo):

FieldTypeDescriptionExample
BoardIDnumberBoard ID1174538260
FWsizenumberFirmware size marker2
RunModenumberRunning mode0
SNstringDevice serial number"2493C45353E0A336"
VersionstringFirmware version"004"
ConnectModenumberConnection mode16
CheckNumstringCheck value"00000000"

Example return structure:

json
{
	"BoardID": 1174538260,
	"FWsize": 2,
	"RunMode": 0,
	"SN": "2493C45353E0A336",
	"Version": "004",
	"ConnectMode": 16,
	"CheckNum": "00000000"
}

Example ​

javascript
import { ServiceKeyboard } from 'wvke-sdk';

async function fetchDeviceInfo() {
	const keyboard = new ServiceKeyboard();

Set Current Config ​

ServiceKeyboard.setConfig()

Brief description: Sets the keyboard's current configuration.

Parameters ​

FieldTypeDescriptionRequired
configIndexnumberConfig index to set, for example 2Yes

Return Value ​

• Overall type: Promise<{ code: number, setConfigName: string, configIndex: number, current: { configName: string, value: number } }> • Description: Returns a Promise. When the operation succeeds, it resolves with the applied result and the current active config information.

Example ​

javascript
async function setCurrentConfig(configIndex) {
	try {
		const result = await ServiceKeyboard.setConfig(configIndex);
		console.log('Config switch result:', result);
		// Example output:
		// {
		//   code: 0,
		//   setConfigName: 'Config2',
		//   configIndex: 2,
		//   current: { configName: 'Config2', value: 2 }
		// }
	} catch (error) {
		console.error('Failed to set config:', error);
	}
}

// Example: switch to config index 2
// setCurrentConfig(2);

Notes ​

TIP

• The config index must be between 1 and 4 • After switching configs, it is recommended to listen to the getCmd event to obtain the result of the config switch • After receiving the config switch success notification, you usually need to fetch the following data again: • Keyboard configuration data • Lighting settings • Key mappings • Other related settings • It is recommended to show a loading state during config switching for a better user experience

Set Polling Rate ​

ServiceKeyboard.setRateOfReturn()

Brief description: Sets the current keyboard polling rate.

Parameters ​

FieldTypeDescriptionRequired
valuestring | numberPolling rate value. Supports protocol values 1 to 7, display values such as 125Hz, 250Hz, 500Hz, 1000Hz, 2000Hz, 4000Hz, 8000Hz, and KHz-style inputs such as 1KHz, 2KHz, 4KHz, 8KHz.Yes

Return Value ​

• Overall type: Promise<{ code: number, key: string, value: number }> • Description: Returns a Promise that resolves to the applied polling-rate result, where key is the display value and value is the protocol value.

Example ​

javascript
async function updateRateOfReturn(value) {
	try {
		const result = await ServiceKeyboard.setRateOfReturn(value);
		console.log('Set polling rate result:', result);
		// Example output: { code: 0, key: '1000Hz', value: 4 }
	} catch (error) {
		console.error('Failed to set polling rate:', error);
	}
}

// Using protocol value
// updateRateOfReturn(4);

// Using display value
// updateRateOfReturn('1000Hz');

// Using KHz style
// updateRateOfReturn('8KHz');

Notes ​

TIP

• In the keyboard firmware protocol, polling rates are represented by values 1 through 7. • The SDK automatically converts common string inputs to the corresponding protocol value. • After setting the polling rate, it is recommended to call ServiceKeyboard.getRateOfReturn() again to confirm the result.

Restore Factory Settings ​

ServiceKeyboard.GFSRestore()

Brief description: Restores the keyboard to factory default settings.

Parameters ​

This method does not require any parameters.

Return Value ​

• Overall type: Promise<{ code: number, success: boolean, message: string }> • Description: Returns a Promise that resolves to the factory-reset result object when the operation succeeds.

FieldTypeDescriptionExample
codenumberAPI status code. 0 means success.0
successbooleanWhether the reset succeeded.true
messagestringResult message."Factory settings restored successfully"

Example ​

javascript
async function restoreFactorySettings() {
	try {
		const result = await ServiceKeyboard.GFSRestore();
		console.log('Factory reset result:', result);
		// Example output: { code: 0, success: true, message: 'Factory settings restored successfully' }
	} catch (error) {
		console.error('Failed to restore factory settings:', error);
	}
}

// restoreFactorySettings();

Notes ​

TIP

• After a factory reset, config slots, lighting parameters, system type, sleep timeout, and other global settings may return to their defaults • After the reset, it is recommended to call getConfig(), getSystemType(), getRateOfReturn(), and getLightingSleepTime() again to refresh the current state • If your app caches key layouts, DPI, or other device-side data, refresh those as well
